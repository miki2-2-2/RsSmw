Immediate¶

SCPI Commands

TRIGger<HwInstance>:FSWeep:IMMediate
class Immediate[source]

Immediate commands group definition. 1 total commands, 0 Sub-groups, 1 group commands

set() → None[source]
# SCPI: TRIGger<HW>:FSWeep:[IMMediate]
driver.trigger.freqSweep.immediate.set()


    INTRO_CMD_HELP: Performs a single sweep and immediately starts the activated, corresponding sweep:

    - FSWeep - RF frequency
    - PSWeep - RF level
    - LFFSweep - LF frequency
    - SWEep - all sweeps
    INTRO_CMD_HELP: Effective in the following configuration:

    - TRIG:FSW|LFFS|PSW|[:SWE]:SOURSING
    - SOUR:SWE:FREQ|POW:MODEAUTO or method RsSmw.Source.LfOutput.Sweep.Frequency.modeAUTO

Alternatively, you can use the IMMediate command instead of the respective SWEep:[FREQ:]|POW:EXECute command.

set_with_opc() → None[source]
# SCPI: TRIGger<HW>:FSWeep:[IMMediate]
driver.trigger.freqSweep.immediate.set_with_opc()


    INTRO_CMD_HELP: Performs a single sweep and immediately starts the activated, corresponding sweep:

    - FSWeep - RF frequency
    - PSWeep - RF level
    - LFFSweep - LF frequency
    - SWEep - all sweeps
    INTRO_CMD_HELP: Effective in the following configuration:

    - TRIG:FSW|LFFS|PSW|[:SWE]:SOURSING
    - SOUR:SWE:FREQ|POW:MODEAUTO or method RsSmw.Source.LfOutput.Sweep.Frequency.modeAUTO

Alternatively, you can use the IMMediate command instead of the respective SWEep:[FREQ:]|POW:EXECute command.

Same as set, but waits for the operation to complete before continuing further. Use the RsSmw.utilities.opc_timeout_set() to set the timeout value.
